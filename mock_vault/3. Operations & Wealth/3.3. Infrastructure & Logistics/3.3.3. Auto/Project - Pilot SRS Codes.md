# Project - Pilot SRS Codes

### System Status: 2005 Honda Pilot
**Active Diagnostic Codes:**
*   **61-10:** Open circuit in the driver's seat belt buckle switch.
*   **85-64:** Inconsistent ECU serial ID (Occupant Detection System).
*   **81-43:** Internal failure of the Passenger Weight Sensor Unit.

---

### 1. Understanding the Codes
*   **Code 61-10 (Driver's Buckle):** The system cannot detect if the driver is buckled. This is often eligible for a **lifetime warranty** at Honda dealerships.
*   **Code 85-64 (OPDS Serial Mismatch):** This occurs when the Occupant Detection System (OPDS) and the SRS unit do not recognize each other’s serial numbers. It often happens if a seat or module was replaced without being "re-initialized."
*   **Code 81-43 (Passenger Weight Sensor):** This indicates a fault in the sensor that detects how heavy a passenger is to determine if the airbag should be active.

### 2. Troubleshooting & DIY Inspection
1.  **Safety First:** Disconnect the battery and wait **3 minutes** before touching any yellow SRS connectors.
2.  **Passenger Seat (85-64 & 81-43):**
    *   Inspect the wiring under the front passenger seat. Ensure the harness isn't pinched in the rails.
    *   Check the connectors for corrosion. A poor connection to the Weight Sensor Unit can trigger an internal failure code.
3.  **Driver's Seat (61-10):**
    *   Inspect the buckle wiring under the driver's seat. 
    *   Clean the buckle switch with a [zero-residue electrical contact cleaner](https://a.co/d/0chWyoQC).

### 3. Warranty & Professional Repair
Honda typically provides a **Lifetime Limited Warranty** on seat belt components (like the buckle for 61-10). The passenger sensor codes (81-43 and 85-64) usually require a dealership-grade scan tool to perform an **OPDS Initialization** or **SWS Calibration**.

### 4. Required Tools for Reset
These codes (specifically the 80-series codes) typically **cannot** be cleared by basic OBD-II readers. They require a scanner capable of bidirectional communication with Honda SRS modules to "marry" the components via initialization.
