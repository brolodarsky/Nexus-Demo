# 🚀 Project: Career System Upgrades

This project outlines the next-level optimizations required to turn the current career strategy into an "ultimate" system, perfectly tailored for the 2026 AI Agent market.

## 1. Application Funnel Tracker
- **The Gap:** The system tracks people (`Professional CRM`) and offers (`Compensation Guide`), but lacks a central Kanban board for tracking the actual *applications* in motion. 
- **The Goal:** Build an `Active Application Pipeline.md` to track the funnel: `[Identified -> Applied -> Phone Screen -> Technical -> Final -> Offer/Rejected]`.

## 2. Public "Proof of Work" (Digital Garden)
- **The Gap:** Sending a recruiter a PDF or a Google Doc is weak. Academic coursework is invisible.
- **The Goal:** Build a high-quality, public-facing personal website (`williamvolodarsky.com`). Automatically publish the best, non-private insights from the `Nexus` vault into a public "Digital Garden" using tools like Obsidian Publish or a Next.js blog template.

## 3. Deliberate Open Source (OSS) Contributions
- **The Gap:** Self-guided projects (`Project Feeder`, `Nexus.0`) don't prove the ability to collaborate with a team of senior engineers in a production codebase.
- **The Goal:** Add an Open Source strategy. Actively target issues in the Agent Orchestration tools currently used (e.g., LangChain, CrewAI, Pydantic). A merged Pull Request in a major AI framework instantly bypasses standard entry-level filters.

## 4. Project-Based Skill Acquisition Schedule
- **The Gap:** "Skills to Get" lists (Kubernetes, Terraform, RAG, Vector DBs) remain untouched unless attached to a real deployment.
- **The Goal:** Tie these skills directly to the next phase of the deployed portfolio.
	- Example: *"Goal for May: Deploy Project Feeder's database to AWS using Terraform"*
	- Example: *"Goal for June: Add a ChromaDB Vector DB to Nexus.0"*

## 5. Peer-Reviewed Mock Interviews
- **The Gap:** The `Interview Prep Hub.md` protocol currently relies on practicing alone.
- **The Goal:** Integrate hostile, high-pressure environments to test STAR stories and technical communication. Utilize platforms like Pramp, interviewing.io, or dedicated tech Discord communities to run mock peer interviews.
