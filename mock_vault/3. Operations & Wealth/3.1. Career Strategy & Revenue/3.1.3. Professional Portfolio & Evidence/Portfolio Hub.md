## 1. Flagship Project: Nexus (Agentic OS)

**The Objective:** Shift the narrative from "I use Obsidian for notes" to "I architected a multi-agentic knowledge system and an Obsidian TypeScript plugin."

### The Elevator Pitch (Interviews/Networking)
> *"For the past year, I’ve been architecting a localized 'Agentic Operating System' called Nexus. Instead of just using consumer AI tools, I built a multi-agent workflow system that processes raw asynchronous mobile captures, evaluates my health metrics, and routes data into a highly structured [Zettelkasten](https://www.google.com/search?q=zettelkasten). Most recently, I engineered a custom TypeScript Obsidian Plugin that utilizes a 'Structure vs. State' architecture to bypass LLM attention degradation (the "Lost in the Middle" phenomenon), turning a static markdown vault into a fully autonomous, auto-updating HUD."*

### Resume Bullet Points (Copy & Paste)
*   **Agentic System Architecture:** Architected 'Nexus', a modular Python orchestration system with 11 agentic skills, 10 slash-command workflows, and 12 deterministic tools, governed by a declarative Agentic Constitution.
*   **Clinical Reasoning Pipelines (GraphRAG):** Designed an "Agentic Doctor Panel" pipeline replacing standard vector search with explicitly routed multi-agent debate to synthesize massive, fragmented longitudinal medical records into actionable clinical insight.
*   **TypeScript / Plugin Development:** Engineered a localized Command Center plugin utilizing a "Structure vs. State" split, managing dynamic Dataview queries via custom CSS and JS injection to create an auto-updating HUD.
*   **Context Optimization:** Designed a dynamic RAG-based instruction loading pipeline to successfully bypass LLM attention degradation ("Lost in the Middle" phenomenon) across massive context windows.
*   **Mobile-to-Edge Deployment:** Established secure, asynchronous YAML/Markdown syncing between mobile capture vectors and the desktop agentic terminal for zero-friction data pipelines.
*   **Agentic File System Navigator (Librarian):** Engineered a stateful LangGraph agent utilizing tree-based navigation, live directory structure injection, and targeted regex grep tools to dynamically navigate 500+ structured markdown notes, replacing vector RAG to solve context fragmentation.

#### Detailed Project Brief: Agentic File System Navigator (Librarian)
> "I engineered a stateful LangGraph agent that dynamically navigates a personal knowledge base of 500+ markdown notes. Unlike standard vector-based RAG which suffers from context fragmentation, the Librarian uses filesystem tools (`read_toc`, `read_note`, `search_vault`) to crawl the vault. It uses tree-based navigation via a live-injected folder map, targeted regex sub-tree grepping, and frontmatter metadata filters, delivering 100% grounded answers cited with Obsidian deep-links."

**Key Technical Achievements:**
- **Context-Preserving Retrieval:** Bypassed vector chunk fragmentation by building tools that allow the agent to read notes in their entirety and trace structured wiki-links.
- **Dynamic State Machine:** Modeled the agent as a LangGraph ReAct architecture with explicit navigation, reading, and self-evaluation nodes.
- **Token & Cost Optimization:** Logged usage metadata per-query, tracking prompt and completion tokens, and implemented cosine distance filtering for semantic search.
- **Privacy-First Infrastructure:** Kept all operations local-first, utilizing API calls solely for the LLM reasoning loop.
